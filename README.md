# Multiton - це шаблон проектування, який є варіантом шаблону Singleton. У шаблоні Multiton кожен екземпляр класу зберігається у колекції, доступ до якої здійснюється за допомогою ключа.

# Призначення Multiton полягає у тому, що при кожному запиті на створення екземпляра перевіряється, чи вже існує екземпляр з відповідним ключем. Якщо такий екземпляр вже існує, повертається посилання на нього; якщо ні, то створюється новий екземпляр, який потім додається до колекції.

# Переваги Multiton полягають у тому, що він дозволяє мати кілька екземплярів класу з різними конфігураціями і забезпечує глобальний доступ до них. Використання Multiton може бути корисним, коли потрібно зберігати обмежену кількість екземплярів класу, але кожен з них має свою унікальну ідентифікацію.

# Adapter - це шаблон проектування, який використовується для забезпечення взаємодії між двома несімісними інтерфейсами. Робота якого полягає в тому, щоб створити посередника, який перетворює інтерфейс одного класу в інтерфейс, зрозумілий іншому класу.

# Призначення Adapter - це забезпечення суміснісності між класами, які мають несумісні інтерфейси, без необхідності змінювати код цих класів. Такий адаптер може бути використаний, коли необхідно використовувати існуючий клас, але його інтерфейс не відповідає потребам.

# Переваги Adapter: - Покращує повторне використання коду; Забезпечує сумісність між різними компонентами системи; Дозволяє використовувати класи з несумісними інтерфейсами разом без необхідності внесення змін в їхній код.

# -------------------------------------------------------------------------------------------------------

# Servant - це структурний шаблон проектування, який дозволяє зменшити кількість взаємодій між об'єктами, перемістивши деякі функції та операції на спеціалізований об'єкт, який називається слугою. Цей об'єкт надає спільні функції або операції для інших об'єктів, що його використовують.

# Призначення Servant полягає в тому, щоб уникнути повторювання коду та підтримати високий рівень повторного використання, перемістивши спільні функції на окремий об'єкт, який використовується як слуга.

# Переваги Servant: Зменшення кількості взаємодій між об'єктами; Зменшення повторного коду і підтримка високого рівня повторного використання; Покращення структури коду та збільшення читабельності.

# -------------------------------------------------------------------------------------------------------

# Messaging - це шаблон проектування, що дозволяє об'єктам спілкуватися один з одним, надсилаючи повідомлення через канал зв'язку. Замість прямого виклику методів інших об'єктів, об'єкти взаємодіють один з одним за допомогою асинхронного обміну повідомленнями.

# Призначення Messaging полягає в тому, щоб знизити повʼязаність обʼєктів, відокремивши їх функціональність та дозволяючи їм взаємодіяти через повідомлення. Кожен об'єкт може відправляти повідомлення іншим об'єктам та обробляти отримані повідомлення.

# Переваги Messaging: Знижує  повʼязаність обʼєктів; Дозволяє створювати більш гнучкі та розширені системи; Підвищує масштабність, оскільки дозволяє легко додавати нові об'єкти та взаємодіяти з існуючими.
